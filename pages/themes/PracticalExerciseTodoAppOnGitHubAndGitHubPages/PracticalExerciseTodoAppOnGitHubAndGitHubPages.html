Collaborate on Todo List Project in GitHub

Collaborate on Todo List Project in GitHub

Introduction

This exercise will guide you through collaborating on a Todo List project using GitHub and deploying it on GitHub Pages.

Create GitHub Repository

Create a new repository on GitHub to host the project code.
Make sure to add a README file, and optionally a .gitignore file

                git init
                git add README.md
                git commit -m "Initial commit"
                git branch -M main
                git remote add origin 
                git push -u origin main

Collaborate Using Branches

Create branches for different features or tasks.
Team members should work on separate branches to avoid conflicts.

                git checkout -b feature/add-todo
                git checkout -b feature/style-todo-list
Once a feature is completed, merge the branch into the main branch.

                git checkout main
                git merge feature/add-todo
                git push origin main

Deploy on GitHub Pages

Deploy on GitHub Pages

Configure GitHub Pages

Go to Repository Settings
Scroll down to the GitHub Pages section under the Code and automation section
Under Source, select the branch you want to deploy from. Typically, this is main or master, and you may select /root to deploy from the root directory or /docs if you have a docs folder.
Click Save to update the settings

Access Your Deployed Site

GitHub Pages will take a few moments to build and deploy your site
After deployment, GitHub Pages will provide a URL for your site, typically in the format: https://<username>.github.io/<repository-name>/

Troubleshooting

Check Build Logs:
If your site isn't deploying, check the build logs in the GitHub Pages section under Settings.
Verify File Paths
Ensure that your file paths are correct and that your HTML, CSS, and JS files are properly referenced.
